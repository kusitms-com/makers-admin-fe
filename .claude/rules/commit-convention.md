# 커밋 컨벤션

커밋 메시지는 Udacity Git Style Guide를 기반으로, 변경 목적을 빠르게 파악할 수 있게 작성합니다.

## 형식

```text
<type>: <subject>
```

scope는 기본적으로 사용하지 않습니다. 여러 목적이 섞이면 커밋을 분리합니다.

## Type

| Type | 용도 |
| --- | --- |
| `feat` | 사용자에게 보이는 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서, 주석, README, AI 지침 변경 |
| `style` | 포맷팅처럼 동작 영향 없는 변경 |
| `refactor` | 동작 변경 없는 코드 구조 개선 |
| `test` | 테스트 추가/수정 |
| `chore` | 설정, 빌드, 의존성, 보조 작업 |

## Subject

- 한국어 명령형 또는 명사형으로 짧게 작성합니다.
- 50자 이내를 우선합니다.
- 무엇을 했는지보다 왜 변경했는지가 드러나게 씁니다.
- 파일명 나열로 subject를 채우지 않습니다.
- 마침표를 붙이지 않습니다.

## 예시

```text
docs: AI 작업 지침 정리
chore: Codex skill proxy 추가
feat: 멤버 승인 목록 API 연결
fix: 프로젝트 수정 폼 파일 전송 오류 수정
test: 후기 목록 empty 상태 테스트 추가
```

## 금지

- `Generated by`, `Co-Authored-By` 같은 AI 출처 문구를 넣지 않습니다.
- 검증하지 않은 변경을 검증한 것처럼 쓰지 않습니다.
- 서로 독립적인 기능과 설정 변경을 한 커밋에 섞지 않습니다.
