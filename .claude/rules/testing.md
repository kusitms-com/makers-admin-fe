# 테스트 규칙

## 테스트 위치

- 단위 테스트와 컴포넌트 테스트는 대상 파일 옆에 둡니다.
- 유틸리티 테스트는 `.test.ts`, React 컴포넌트 테스트는 `.test.tsx`를 사용합니다.
- E2E 테스트는 `e2e`에 둡니다.

## 도구

- 단위/컴포넌트 테스트는 Vitest를 사용합니다.
- React 동작 테스트는 Testing Library를 사용합니다.
- 컴포넌트 테스트 환경은 happy-dom을 사용합니다.
- E2E flow는 Playwright를 사용합니다.

## 테스트 대상

- 회귀 가능성이 있는 동작을 테스트합니다.
- API hook은 테스트 패턴이 있을 때 query/mutation 동작을 테스트합니다.
- form은 검증과 submit 동작을 테스트합니다.
- 관리자 상태 변경은 화면상의 상태 전환과 query invalidation 동작을 테스트합니다.

## 피할 것

- 인터랙티브 UI에 snapshot-only 테스트만 두지 않습니다.
- 구현 세부사항을 그대로 복제하는 테스트를 피합니다.
- 집중된 컴포넌트/단위 테스트로 증명 가능한 변경에 광범위한 E2E 테스트를 쓰지 않습니다.
