# 검증 규칙

주장을 증명할 수 있는 가장 작은 검증을 선택합니다.

## 명령어 선택

- 문서만 변경한 경우: 변경 파일을 확인하고, formatting 또는 생성물이 관련되지 않으면 build는 생략합니다.
- 좁은 컴포넌트/훅 변경: 관련 테스트가 있으면 실행합니다.
- 공통 유틸리티/API 변경: 관련 테스트와 `pnpm lint`를 실행합니다.
- 앱 전반 변경: `pnpm lint`와 `pnpm build`를 실행합니다.
- E2E 영향이 있는 flow: 필요한 경우 특정 Playwright 테스트 또는 `pnpm test:e2e`를 실행합니다.

## 보고

최종 보고에는 아래 내용을 포함합니다.

- 무엇을 변경했는지
- 무엇을 검증했는지
- 검증하지 못한 항목과 이유

명령어가 환경 또는 의존성 문제로 실패하면 blocker와 다음으로 신뢰할 수 있는 근거를 보고합니다.
