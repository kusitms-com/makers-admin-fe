# commit-kr

커밋 메시지만 제안합니다. `git add`와 `git commit`은 직접 실행하지 않습니다.

## Workflow

1. `git status --short`로 변경 파일을 확인합니다.
2. `git diff`와 `git diff --staged`를 확인합니다.
3. `.claude/rules/commit-convention.md`를 읽습니다.
4. 변경 목적을 기능/수정/문서/테스트/설정/리팩터링으로 분류합니다.
5. 변경 목적이 여러 개면 커밋 분리를 제안합니다.
6. 아래 형식으로 명령을 출력합니다.

## Format

```bash
git add <파일...>
git commit -m "<type>: <subject>"
```

여러 커밋이 적합하면 파일 묶음별로 나눠 제안합니다.

```text
커밋 1: docs: AI 작업 지침 정리
- AGENTS.md
- CLAUDE.md
- .claude/**

커밋 2: chore: 기본 소스 폴더 추가
- src/api/.gitkeep
- src/hooks/.gitkeep
```

## Type

- `feat`: 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 포맷팅, 세미콜론 등 동작 영향 없는 변경
- `refactor`: 기능 변화 없는 코드 구조 변경
- `test`: 테스트 추가/수정
- `chore`: 설정, 빌드, 보조 작업

## Rules

- subject는 한국어 명령형으로 작성합니다.
- subject는 50자 이내를 우선합니다.
- scope는 사용하지 않고 `type: subject` 형식을 사용합니다.
- 상세 규칙은 `.claude/rules/commit-convention.md`를 따릅니다.
- 커밋 메시지는 변경 이유가 드러나게 작성하고, 파일명 나열만 하지 않습니다.
- 미검증 변경이 있으면 커밋 전 확인 필요 항목으로 표시합니다.
- `Co-Authored-By`, `Generated by` 같은 AI 출처 문구를 넣지 않습니다.
